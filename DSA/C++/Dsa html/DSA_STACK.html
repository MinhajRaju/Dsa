📖 Stack — কী, কেন, কখন?
💡 মূল ধারণা

Stack মানে থালার স্তূপ। সবার উপরে রাখা → সবার আগে নেওয়া। এটাই LIFO (Last In, First Out)।


মূল operations: push(x) — উপরে রাখো। pop() — উপর থেকে নাও। top() — উপরেরটা দেখো।


Key insight: আগের state remember করতে বা নেস্টেড structure undo করতে stack সবচেয়ে কার্যকর।

🎯 কখন ব্যবহার করবো?
  • Bracket matching → push open, pop on close
  • Undo/redo → state stack এ রাখো
  • Nested structures → recursion replace
  • Expression evaluation → operator/operand
  • DFS traversal → recursion stack
  • যখন সাম্প্রতিক element বারবার দরকার হয়
🔍 এই signal দেখলে Stack ভাবো
📌 "valid brackets / parentheses" → push/pop
📌 "undo last operation" → pop
📌 "evaluate expression" → operator stack
📌 "decode/expand nested string"
📌 "cancel/remove previous element"
📌 "last seen" বা "previous element" দরকার

📖 Monotonic Stack — কী, কেন, কখন?
💡 মূল ধারণা

Monotonic Stack মানে এমন stack যেটা সবসময় increasing বা decreasing order maintain করে।


নতুন element ঢোকানোর আগে stack এর top এর চেয়ে ছোট/বড় element গুলো pop করে দেওয়া হয়।


Key: pop হওয়ার সময়ই answer calculate হয়। O(n) — প্রতিটি element একবার push, একবার pop।

🎯 কখন ব্যবহার করবো?
  • Next Greater Element → decreasing stack
  • Next Smaller Element → increasing stack
  • Previous Greater/Smaller → same idea, left→right
  • Largest Rectangle → সব bars এর span
  • Stock Span → days since last bigger price
  • যখন nearest bigger/smaller দরকার O(n) এ
🔍 এই signal দেখলে Monotonic Stack ভাবো
📌 "next greater / smaller element"
📌 "previous greater / smaller element"
📌 "daily temperatures" wait days
📌 "stock span" consecutive days
📌 "largest rectangle" in histogram
📌 "subarray min/max" contribution
