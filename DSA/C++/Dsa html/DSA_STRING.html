📖 String — কী, কেন, কখন?
💡 মূল কৌশলগুলো

Two Pointer: l=0, r=n-1। দুই দিক থেকে ভেতরে আসো। Palindrome check, reverse, vowel swap।


Expand Around Center: প্রতিটি index কে center ধরে বাইরে expand করো। Longest palindromic substring।


Stack: LIFO — bracket matching, decode nested string।


DP (2D table): Subproblem overlap থাকলে। Edit distance, regex, wildcard matching।

🎯 কখন কোন technique?
  • Two Pointer — palindrome check, reverse, opposite ends
  • Expand Center — longest palindromic substring O(n²)
  • Stack — nested structure decode, valid parentheses
  • DP [i][j] — edit distance, regex, wildcard, LCS
  • RLE scan — count & say, string compression
  • KMP prefix — pattern matching, shortest palindrome
🔍 এই signal দেখলে কী ভাবো?
📌 "palindrome check" → two pointer l,r
📌 "longest palindrome" → expand around center
📌 "decode k[...]" → stack + counter
📌 "min edit operations" → edit distance DP
📌 "match with . and *" → regex DP table
📌 "compress aabbb→a2b3" → RLE two pointer
⚡ তিনটি Main Pattern
① Two Pointer
l=0, r=n-1
while l < r:
  if s[l]!=s[r]:
    return false
  l++; r--
return true

Palindrome Check

② Expand Center
best=""
for c in 0..2n-1:
  l=c/2,r=l+c%2
  while s[l]==s[r]:
    update best
    l--; r++

Longest Palindrome

③ Edit Distance DP
dp[i][j]=
 match? dp[i-1][j-1]
 else 1+min(
  dp[i-1][j],
  dp[i][j-1],
  dp[i-1][j-1])

Min Operations

Easy— two pointer, frequency count, basic scan
Medium— expand center, stack decode, RLE
Hard— 2D DP, KMP, advanced pattern

📖 Sliding Window on String — কী, কেন, কখন?
💡 মূল ধারণা

Sliding Window মানে string এর উপর একটা "জানালা" সরানো। Window এর ভেতরের character frequency efficiently maintain করো।


Fixed Window: size সবসময় k। Right এ যোগ, Left এ বাদ।


Variable Window: condition মেটানোর জন্য expand বা shrink। Min/Max window খোঁজো।

🎯 কখন কোন technique?
  • Fixed window freq — permutation in string, anagram
  • Variable window — longest no-repeat, min window
  • HashMap freq — first unique, ransom note
  • Bijection map — isomorphic strings, word pattern
  • Sort key grouping — group anagrams
  • need/have two maps — minimum window substring
🔍 Signal দেখলে কী ভাবো?
📌 "longest without repeat" → variable window + set
📌 "permutation/anagram in s" → fixed window freq
📌 "min window containing all" → shrink window
📌 "group by anagram" → sorted key map
📌 "first unique char" → freq map scan
📌 "char-to-char mapping" → bijection double map

③ Min Window
need=freq(t)
have=0, formed=0
for r in 0..n:
  add s[r]
  while formed==req:
    update ans
    remove s[l]; l++

Minimum Window Substr

Easy— frequency map, bijection, linear scan
Medium— sliding window, fixed frequency window
Hard— variable shrink, multi-map, word-level
